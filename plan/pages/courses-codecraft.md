## Paths

| Path | Role focus | Example stack & outcomes | **Tools you’ll learn & use** |
| :--- | :--- | :--- | :--- |
| **Frontend developer** | Web UI, accessibility, performance | HTML/CSS/JS, React or Vue, responsive apps → production-style UIs | **Cursor** or **Visual Studio Code**; **Git** + **GitHub**; **Chrome DevTools**; **npm** / **pnpm**; **Figma** (handoff) optional |
| **Backend developer** | APIs, data, auth, server logic | Node or Python, REST/GraphQL, SQL/NoSQL → services + docs | **Cursor** / **VS Code**; **Postman** or **Insomnia**; **Docker** (local services); **PostgreSQL** / **MongoDB** tools; **Git** |
| **Full-stack developer** | End-to-end product delivery | Frontend + backend + deployment → 2–3 full apps | Above + **Vite** / framework CLI; one **hosting** target (e.g. **Vercel**, **Netlify**, **Railway**, or **Render**) — pick what you teach |
| **Mobile developer** | Cross-platform apps | Flutter or React Native → installable builds | **Android Studio**; **VS Code** or **Cursor**; **emulators** + physical **Android** device; **Xcode** only if iOS module overlaps |
| **iOS developer** | Native Apple platforms | Swift, SwiftUI, Xcode → App Store–quality trajectory | **Xcode**; **Swift** / **SwiftUI**; **Simulator** + physical **iPhone**; **TestFlight** workflow as advanced topic |
| **DevOps** | Delivery, reliability, cloud | CI/CD, containers, cloud → pipelines + deployed infra | **Git** + **GitHub Actions** (or **GitLab CI**); **Docker** + **Docker Compose**; **cloud console** (e.g. **AWS**, **GCP**, or **Azure** — pick one primary); **SSH**; basics of **observability** (logs/metrics tool of choice) |

## IDE note

Teaching **Cursor** (or **Copilot**-style assist) alongside **VS Code** signals modern employability; clarify **student license** vs **free tier** expectations on the enroll page.

## Site UX notes

- **Courses page:** One **CodeCraft** hero block + **tabs or accordions** per path (syllabus, duration, prerequisites, next cohort).
- **Enroll:** Dropdown or step — “Software development → choose path” (optional second path / upgrade later).
- **Duration:** Typically **8–14+ weeks per path** depending on depth; **full stack** often longest; offer **foundations** module first if beginners are common.
- **Bundles:** e.g. Frontend + Backend → Full-stack discount; Mobile + Backend for API-heavy apps.
